# SquadBattleTD — Owner Decisions Required

> Version: 0.1 | Date: 2026-03-28  
> **For:** Morbror (product owner)  
> **From:** Architecture review

This document lists every decision that cannot be made by the developer alone. Each item is either a business decision, a creative direction call, or something that requires access, budget, or taste that only the product owner can provide.

Nothing in this list is blocking Phase 0 or Phase 1 — but the further you get without answers, the more rework you risk.

---

## 🌐 Infrastructure & Domain

### 1. Domain Registration (sbtd.io)
**Status:** Blocking for Phase 6 (alpha release)  
**What's needed:**
- Someone needs to register `sbtd.io` via Cloudflare Registrar (recommended — integrates directly with Cloudflare Tunnel, no nameserver change needed)
- Requires a Cloudflare account with a payment method
- Cost: ~$10–15/year for a .io domain

**Question:** Does Morbror want this under their Cloudflare account, or should a shared account be created for the project?

---

### 2. Cloudflare Tunnel Setup
**Status:** Blocking for Phase 0 end-state (public URL)  
**What's needed:**
- Cloudflare account to create the tunnel (free tier is fine)
- Tunnel credentials file (`credentials.json`) — generated by Cloudflare CLI, deployed as a k8s Secret
- Can use a temporary domain (e.g., `*.trycloudflare.com`) for Phase 0 testing without owning `sbtd.io`

**Question:** Who owns the Cloudflare account for this project?

## 🎨 Art & Visual Direction

### 3. Sprite / Art Style
**Status:** Not blocking Phase 1, but direction needed before Phase 5 polish  
**Current plan:** Colored rectangles as placeholder sprites during development.

**Decisions needed:**
- **Style:** Pixel art? Vector/flat? Hand-drawn? 3D-rendered-to-sprite?
- **Resolution:** 16×16, 32×32, 64×64 tiles?
- **Color palette:** Restricted palette (retro feel) or full color?
- **Tone:** Cute/cartoony or gritty/dark?
- **Scope for alpha:** Placeholder-quality pixel art is acceptable — but even placeholders should have consistent style

**Questions:**
- Is Morbror handling art personally, or will an artist be hired?
- If hiring: what's the budget and timeline?
- Can we use royalty-free asset packs (e.g., Kenney.nl) for alpha, then commission originals later?

---

### 4. Sound Design
**Status:** Not in alpha scope, but decide early if it affects art style  
**Questions:**
- Is audio in scope for alpha at all?
- Chiptune / 8-bit to match pixel art? Orchestral?
- Free sound libraries (freesound.org) acceptable for alpha?

## ⚔️ Game Design & Balance

### 5. Game Balance (Unit Stats)
**Status:** Can prototype with placeholder numbers, but needs owner review before alpha  
**What this affects:** Cost, HP, damage, speed, attack rate for every unit. The numbers define the game feel.

**Questions:**
- Who owns balance decisions — is this a collaborative process or developer-driven with owner feedback?
- What's the process for playtesting balance? (Sessions needed)
- Are any numbers carried over from Legion TD Web prototype, or starting fresh?

---

### 6. Map Design
**Status:** Need at least 1 map for Phase 1  
**Decisions needed:**
- Number of lanes (1 lane for simplicity? 2 lanes for strategy depth?)
- Map size (small = faster games, large = more strategic placement)
- Is the opponent's side a mirror image, or a different layout?
- How many maps for alpha? (1 is fine)
- Who draws the layout? (Grid sketch on paper is enough for the developer to implement)

**Suggestion:** Start with a single 1-lane map, ~20×12 tiles. Owner draws a rough sketch, developer implements.

---

### 7. Race Design
**Status:** Need at least 1 race for Phase 1, 2 races recommended for alpha  
**Current assumption:** Human Alliance as Race 1.

**Decisions needed:**
- What is Race 2? (Undead? Orc? Elf? Something original?)
- What's the thematic/mechanical identity of each race?
  - e.g., Human Alliance = balanced generalists
  - e.g., Undead = high damage, low cost, fragile
- How many units per race? (6–8 recommended for alpha)
- Are race abilities (e.g., racial bonuses) in scope for alpha?

**Questions:**
- Is this inspired by existing Warcraft 3 races, or fully original?
- Any lore or world-building direction?

---

### 8. Game Mode Priority
**Status:** Affects Phase 1–4 development order  
**Options:**
- **1vCPU first** (simpler, can play alone, great for balance testing)
- **1v1 first** (more fun, requires two players for testing)
- **Both simultaneously** (more work, but the systems overlap heavily)

**Recommendation:** Build combat engine (Phase 2) as 1vCPU by default — use a dummy CPU that just sends random units. Then Phase 3 adds real multiplayer. This lets testing happen solo.

**Question:** Does Morbror agree with this priority, or is 1v1 human play the primary alpha target?

## 🎯 Release Criteria

### 9. Alpha Definition — "Good Enough"
**Status:** Needed before Phase 6 planning  
**Questions:**
- What does "alpha" mean for this project? Playable by friends? Public-facing? Invite-only?
- Minimum viable feature set — is one race + one map + 1vCPU enough for alpha?
- Is there a target number of simultaneous players the Pi needs to support? (Estimate: Pi 4 can handle ~10–20 concurrent game rooms)
- Bug tolerance: what severity is acceptable at alpha launch?
- Any hard deadline? (e.g., "want to show friends by summer 2026")

---

**Status:** Decide before Phase 5  
**Questions:**
- Artist: hire, use AI-generated sprites, or free assets for alpha?
- Sound designer: in scope for alpha?
- If hiring anyone: what's the budget?
