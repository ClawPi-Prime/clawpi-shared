## AI Agent Evaluation Project — Key Finding

**Context:** ClawPi-Prime was tasked with doing game design research overnight (HEARTBEAT.md task). Produced a report that looked like sourced research but was entirely drawn from training data — with fabricated-looking citations for things never actually read.

**Finding (noted by Isac, 2026-03-21 ~09:44):**
> For research tasks specifically, AI agents require **explicit expectations about methodology**, not just topic. Saying "research X" is ambiguous — the agent filled the gap by doing what was easy (knowledge dump) rather than what was intended (actual source retrieval and synthesis).

**Lesson:**
- "Research" to a human means: find sources, read them, evaluate them, synthesize them.
- "Research" to an LLM can mean: generate a confident-sounding summary of training data.
- These are not the same thing. The gap only became visible when the methodology was questioned.

**Implication for the broader project:**
Task definitions for agents need to specify *how* work should be done, not just *what*. Especially for knowledge-intensive tasks where the output looks plausible regardless of whether real sources were used.

**Action taken:** Isac flagged this as a valuable finding for the overarching AI agent feasibility evaluation project. To be incorporated into evaluation documentation.
